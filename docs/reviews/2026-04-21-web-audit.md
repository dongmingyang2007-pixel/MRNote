## Executive Summary

This audit found a real mix of UX regressions, authorization bugs, cross-workspace data exposure paths, and a few remaining low-severity product gaps versus the notebook spec.

The highest-risk confirmed issues are now fixed in the worktree:

- cross-workspace study-file ingestion
- billing actions exposed to non-admin workspace members
- lateral AI action metadata leakage
- Google OAuth initiation CSRF
- private notebook visibility missing on server-side reads/search/home
- attachment links usable as trusted active-content hosts
- first-run auth/onboarding flow regressions
- unified search returning study documents that the UI could not open

What remains open after this round is much smaller and mostly low severity:

- study uploads have been broadened substantially, but still do not satisfy the spec's literal “any file” scope

## Fixed In This Audit

### Backend security

1. `Fixed` Cross-workspace study ingestion no longer trusts arbitrary `data_item_id`.
   - `apps/api/app/routers/study.py`
   - `apps/api/app/services/study_pipeline.py`
   - Result: users cannot ingest another workspace's private uploaded file into their own notebook.

2. `Fixed` Billing checkout and portal actions are now restricted to `owner/admin`.
   - `apps/api/app/routers/billing.py`
   - Result: workspace members can no longer mutate billing for the whole workspace.

3. `Fixed` AI action summaries are filtered for non-privileged members.
   - `apps/api/app/routers/ai_actions.py`
   - `apps/api/app/routers/notebooks.py`
   - `apps/api/app/services/search_dispatcher.py`
   - Result: members no longer see other users' AI output summaries, notebook titles, and page metadata through home/search/list endpoints.

4. `Fixed` Google OAuth authorization now enforces same-origin initiation.
   - `apps/api/app/routers/auth.py`
   - Result: cross-site login/connect initiation is no longer CSRF-able through top-level navigation alone.

5. `Fixed` Private notebook visibility is now enforced server-side.
   - `apps/api/app/routers/notebooks.py`
   - `apps/api/app/routers/search.py`
   - `apps/api/app/routers/study.py`
   - `apps/api/app/routers/attachments.py`
   - Result: private notebooks/pages/study assets/attachments no longer leak to ordinary workspace members.

6. `Fixed` Attachment downloads no longer expose bare inline presigned URLs.
   - `apps/api/app/routers/attachments.py`
   - `apps/api/app/services/storage.py`
   - Result: uploaded HTML/SVG cannot be trivially hosted as trusted first-party active content via attachment links.

7. `Fixed` Unified `files` search now includes study documents, not only page attachments.
   - `apps/api/app/services/search_dispatcher.py`

### Frontend UX / product correctness

1. `Fixed` Registration code flow now transitions into the code-entry step after `/api/v1/auth/send-code` succeeds.
   - `apps/web/app/[locale]/(auth)/register/page.tsx`

2. `Fixed` Onboarding no longer marks itself complete when notebook/page creation fails, and notebooks home refreshes immediately after onboarding seeds data.
   - `apps/web/components/onboarding/OnboardingWizard.tsx`
   - `apps/web/hooks/useOnboardingStatus.ts`
   - `apps/web/app/[locale]/workspace/notebooks/page.tsx`

3. `Fixed` Notebook workspace now boots as a bare canvas instead of auto-opening the pages side panel.
   - `apps/web/components/console/NotebookSidebar.tsx`
   - aligns with spec §6.3 lines `469-519`

4. `Fixed` Locale-aware notebook navigation now preserves `/en/...` instead of dropping back to default-locale paths.
   - `apps/web/app/[locale]/workspace/notebooks/page.tsx`
   - `apps/web/app/[locale]/workspace/notebooks/[notebookId]/page.tsx`

5. `Fixed` Sidebar page titles now refresh after note renames.
   - `apps/web/components/notebook/contents/NoteWindow.tsx`

6. `Fixed` Search window now renders `files` and `recent AI actions` groups with working i18n copy.
   - `apps/web/components/notebook/contents/SearchWindow.tsx`
   - `apps/web/messages/en/console-notebooks.json`
   - `apps/web/messages/zh/console-notebooks.json`
   - aligns with spec §12.1-12.3 lines `915-940`

7. `Fixed` Search `files` hits for study documents can now actually open instead of dead-ending on missing `attachment_id`.
   - `apps/web/components/notebook/contents/SearchWindow.tsx`
   - `apps/web/components/notebook/contents/FileWindow.tsx`
   - `apps/web/components/console/editor/extensions/FileBlock.tsx`

8. `Fixed` Study workspace now falls back to notebook page slugs when `metadata_json` is delayed or missing, so generated overview/notes/chapter pages still appear.
   - `apps/web/components/notebook/contents/StudyWindow.tsx`
   - aligns with spec §10.0-10.2 lines `824-858`

9. `Fixed` Study uploads now accept arbitrary study files, not only a narrow document whitelist, and unknown binaries fall back to metadata-based ingestion instead of failing.
   - `apps/api/app/services/upload_validation.py`
   - `apps/api/app/services/document_indexer.py`
   - `apps/api/app/services/study_pipeline.py`
   - `apps/web/lib/study-upload.ts`
   - `apps/web/components/notebook/contents/StudyWindow.tsx`
   - `apps/web/components/notebook/contents/study/AssetsPanel.tsx`
   - Added support now includes unrestricted file selection in the web app, generic upload acceptance for unknown extensions, and fallback indexing for non-text / non-previewable files via safe file metadata and readable previews when possible.
   - `svg/svgz` remain intentionally blocked as active-content exceptions.
   - aligns materially with spec §10.0-10.2 lines `824-850`

10. `Fixed` Login failure feedback no longer self-clears after ~1 second.
   - Root cause: public auth `401` was incorrectly treated as session expiry.
   - `apps/web/lib/api.ts`
   - `apps/web/app/[locale]/(auth)/login/page.tsx`

11. `Fixed` Chinese auth screens no longer leak raw English backend errors for common auth failures.
    - `apps/web/lib/auth-errors.ts`
    - `apps/web/app/[locale]/(auth)/login/page.tsx`
    - `apps/web/app/[locale]/(auth)/register/page.tsx`
    - `apps/web/app/[locale]/(auth)/forgot-password/page.tsx`
    - `apps/web/messages/en/auth.json`

12. `Fixed` Root-level global CSS ownership was centralized so notebook surfaces no longer emit the unused `not-found.css` preload warning.
    - `apps/web/app/layout.tsx`
    - `apps/web/app/[locale]/layout.tsx`
    - `apps/web/app/not-found.tsx`

13. `Fixed` Notebook home now behaves as a notebook OS home, not only a card list.
    - `apps/web/app/[locale]/workspace/notebooks/page.tsx`
    - current modules include notebooks, continue writing, recommended, recent study, AI today, work themes, long-term focus
    - aligns substantially with spec §6.2 lines `457-467`

14. `Fixed` Product copy on onboarding/marketing was shifted from consultant/client framing to notebook AI workspace framing.
    - `apps/web/messages/en/onboarding.json`
    - `apps/web/messages/zh/onboarding.json`

15. `Fixed` Study workspace now exposes a notebook-scoped progress and weekly-learning surface instead of only upload / assistant / deck tools.
    - `apps/api/app/routers/study.py`
    - `apps/api/app/schemas/study.py`
    - `apps/api/app/services/study_insights.py`
    - `apps/web/components/notebook/contents/StudyWindow.tsx`
    - `apps/web/components/notebook/contents/study/StudyProgressPanel.tsx`
    - `apps/web/messages/en/console-notebooks.json`
    - `apps/web/messages/zh/console-notebooks.json`
    - Surface now includes recent 7-day study activity, review pressure by deck, weak-card surfacing, recent study actions, and direct next moves back into notes / assistant / review.
    - aligns materially with spec §10.2 / §10.5 lines `859-885`

## Remaining Open Issues

- No active runtime bugs remain from this audit scope.
- Residual gap is now policy-level rather than implementation-level:
  - `svg/svgz` uploads stay blocked on purpose as active-content exceptions even though the product spec says “any file”
  - if the product wants literal no-exception intake, the security posture needs to be revised explicitly
